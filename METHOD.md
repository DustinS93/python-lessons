# METHOD.md — How We Work

## Philosophy
Breadth of knowledge, depth of skill.
Shell drills for familiarity, puzzles for building.

## Shell Drills
- Do these before puzzles on new concepts
- Guess what it returns before running it
- Test one thing at a time
- Use python3 interactive shell

## Puzzles
- Struggle first — ask second
- Reference your own previous puzzles, go look them up yourself
- AI explains concepts, never writes the code
- If you looked something up, understand it before moving on
- Commit after every puzzle, no exceptions

## Debugging Rules
- Read the error message before asking for help
- Try to fix it yourself first
- Don't just accept a fix — understand why it works

## Git Habits
- Commit after every finished puzzle
- Push before moving to next puzzle
- Meaningful commit messages describing what you built
